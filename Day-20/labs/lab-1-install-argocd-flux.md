# 🧪 Lab 1: Installing ArgoCD and Flux v2

In this lab, we will install the control planes for both ArgoCD and Flux v2. This forms the infrastructure baseline of our GitOps system.

---

## Part A: Installing ArgoCD

ArgoCD is usually installed as a set of deployments inside a dedicated namespace.

### Step 1: Create Namespace and Apply Manifests
We will use the official community installation manifests:

```bash
# Create the namespace
kubectl create namespace argocd

# Apply the non-high-availability manifests (suitable for dev/test)
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

### Step 2: Verify ArgoCD Control Plane Pods
Wait for all pods in the `argocd` namespace to reach the `Running` state:

```bash
kubectl get pods -n argocd
```

**Expected Output:**
```text
NAME                                  READY   STATUS    RESTARTS   AGE
argocd-application-controller-0       1/1     Running   0          45s
argocd-applicationset-controller-...  1/1     Running   0          45s
argocd-dex-server-54d9c79f97-abcde    1/1     Running   0          45s
argocd-notifications-controller-...   1/1     Running   0          45s
argocd-redis-8c46445dc-abcde          1/1     Running   0          45s
argocd-repo-server-799ffbc75-abcde    1/1     Running   0          45s
argocd-server-57f955d888-abcde        1/1     Running   0          45s
```

### Step 3: Access the Web UI
By default, the ArgoCD API Server is not exposed externally. We will use port-forwarding to access it locally:

```bash
kubectl port-forward svc/argocd-server -n argocd 8080:443
```
*Leave this terminal window running, and open a new terminal window for subsequent commands.*

Open your web browser and navigate to: `https://localhost:8080`.
*(Ignore the SSL certificate warnings, as it is a self-signed dev certificate.)*

### Step 4: Retrieve the Admin Password
The initial password for the admin account is autogenerated and stored in a Kubernetes secret:

```bash
# Retrieve and decode the secret
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 --decode; echo
```
Log in to the Web UI using:
* **Username:** `admin`
* **Password:** *(the string returned by the base64 command)*

## Part B: Setting up Flux v2

Flux uses a CLI to perform bootstrap operations and check compatibility.

### Step 1: Install the Flux CLI
Choose the command matching your local OS:

* **macOS / Linux (Homebrew):**
  ```bash
  brew install fluxcd/tap/flux
  ```
* **Linux (Direct script):**
  ```bash
  curl -s https://fluxcd.io/install.sh | sudo bash
  ```
* **Windows (Chocolatey / Scoop):**
  ```powershell
  choco install flux
  # or
  scoop install flux
  ```

### Step 2: Verify CLI and Cluster Compatibility
Flux requires certain API versions to exist in the Kubernetes cluster. Run the precheck utility to verify if your cluster is supported:

```bash
flux check --pre
```

**Expected Output:**
```text
► checking prerequisites
✔ Kubernetes 1.29.2 >=1.26.0
✔ prerequisites checks passed
```
If all checks pass, your cluster is ready to be bootstrapped with Flux controllers.

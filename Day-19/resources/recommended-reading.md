# 📚 Day 19 Resources & Recommended Reading

Expand your SRE knowledge and debugging toolkit with these selected references, books, and tools.

---

## 1. Kubernetes Troubleshooting Tools

Make sure you explore and integrate these open-source SRE tools into your daily workflow:

*   **[Stern](https://github.com/stern/stern):** Multi-pod log tailing for Kubernetes. Allows you to stream logs from multiple containers matching a regular expression.
*   **[K9s](https://github.com/derailed/k9s):** A terminal-based UI to interact with and manage your Kubernetes clusters. Excellent for quick troubleshooting.
*   **[Netshoot](https://github.com/nicolaka/netshoot):** The developer's network trouble-shooting swiss-army knife container for Kubernetes.
*   **[Kube-shark](https://github.com/kubeshark/kubeshark):** API traffic analyzer for Kubernetes. Provides real-time visibility into all network traffic.

---

## 2. Recommended Reading & Blogs

*   **[Kubernetes Official Debugging Guide](https://kubernetes.io/docs/tasks/debug/):** The official entry point for application troubleshooting, node debugging, and DNS verification.
*   **[Understanding Linux Container Exit Codes](https://tldp.org/LDP/abs/html/exitcodes.html):** The fundamental reference for standard shell exit codes.
*   **[CoreDNS Performance Tuning](https://coredns.io/plugins/cache/):** Learn how caching rules impact core metrics and latency.
*   **[A Guide to cgroups v2 in Kubernetes](https://kubernetes.io/docs/concepts/architecture/cgroups/):** Discover how cgroups govern CPU and memory allocations in modern clusters.

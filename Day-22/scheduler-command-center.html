💡 Interactive Scheduling Lab Walkthroughs

This dashboard simulates exactly how the Kubernetes scheduler filters and scores nodes. Try the quick scenarios below to see how constraints impact scheduler decisions:

Scenario 1

Taint & Toleration Routing

Load the GPU ML Trainer template. Notice it has a toleration for hardware=gpu. Step-schedule it and watch it land on the tainted worker-gpu-01 node.

Scenario 2

High Availability Spreading

Load the Web Frontend template and queue 4 pods. Click schedule. Host anti-affinity will ensure no two frontend replicas land on the same VM node.

Scenario 3

Bin Packing vs. Spreading

Add 3 default pods. Switch the Strategy above to **Bin Pack**. Run them and observe the scheduler packing pods onto worker-01 before utilizing others.
